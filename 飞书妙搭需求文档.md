# 上市公司高管信息搜索器 — 飞书妙搭应用需求文档

## 一、应用概述

### 1.1 应用目标

搭建一款内部使用的上市公司高管信息搜索工具。用户（投资机构从业人员）通过多维筛选条件，快速找到符合条件的上市公司高管候选人，并将结果导出为 Excel 文件。

### 1.2 使用场景

- 寻访特定行业、特定市值区间的上市公司高管
- 按职位（如 CFO、董事长）搜索候选人名单
- 筛选具有特定背景（如会计师事务所/投行经历）的高管
- 搜索结果导出后用于猎头外发或内部存档

### 1.3 数据规模

- 当前数据量：约 **34,219 条**高管记录
- 覆盖公司数：约 **7,588 家**上市公司
- 数据来源：A 股（沪深主板、科创板、创业板、北交所）+ 港股（约 3,166 只）
- 数据字段：每条记录包含高管姓名、公司、职位、行业、市值、学历、生日、任职时间等
- 未来扩展：接入 Capital IQ 等第三方 API，补充更多维度数据

### 1.4 数据接入方式

- **数据源**：外部 REST API（iFinD 同花顺金融数据接口 + 新浪行情接口）
- **更新方式**：定时调用 API 自动更新，不手动上传
- **存储方式**：飞书多维表格（Bitable）作为数据存储层，或通过 API 直连后端数据库

## 二、数据结构

### 2.1 核心数据表：高管信息表

每条记录代表一位高管在一家公司的任职信息。一人可在多家公司任职，对应多条记录。

| 字段名 | 字段类型 | 说明 | 示例值 |
|--------|----------|------|--------|
| `name` | 文本 | 高管姓名 | 张三 |
| `company` | 文本 | 公司全称 | 平安银行股份有限公司 |
| `code` | 文本 | 股票代码 | 000001、0700.HK |
| `exchange` | 单选 | 上市板块 | 主板、科创板、创业板、北交所、港股 |
| `position` | 文本 | 职位名称 | 董事长、总经理、CFO、独立董事 |
| `industry` | 单选 | 一级行业（申万分类） | 银行、非银金融、医药生物 |
| `subIndustry` | 单选 | 二级行业（申万分类） | 国有大型银行、股份制银行 |
| `marketCap` | 数字 | 市值（亿，本币） | 2078.4 |
| `marketCapCurrency` | 单选 | 市值币种 | CNY（人民币）、HKD（港元） |
| `education` | 文本 | 最高学历 | 博士研究生、MBA |
| `birthday` | 文本 | 出生年份（格式不统一，用文本存储） | 1968、1968-03 |
| `tenure` | 文本 | 任职起始时间（格式不统一，用文本存储） | 2019-03 |
| `listingYear` | 数字 | 公司上市年份 | 1991 |
| `province` | 文本 | 公司注册省份 | 广东省、北京市 |
| `gender` | 单选 | 性别（当前数据暂缺，字段预留） | 男、女 |
| `hasCPABackground` | 布尔 | 是否有会计师事务所经历（当前数据暂缺，字段预留） | 是/否 |
| `hasIBBackground` | 布尔 | 是否有投行/券商经历（当前数据暂缺，字段预留） | 是/否 |
| `isCurrentlyListed` | 布尔 | 当前是否任职于上市公司（用于统计"上市任比例"） | 是/否 |

> **字段说明**：
> - 港股市值单位为亿港元（HKD），A 股为亿人民币（CNY），展示时需区分标注
> - 港股市值通过新浪行情接口获取（价格 × 总股本），覆盖率约 45%，无数据时显示"—"
> - `gender`、`hasCPABackground`、`hasIBBackground` 三个字段当前暂无数据，预留供未来 Capital IQ 等接口补充；**当前版本无需为这三个字段提供筛选入口**
> - `isCurrentlyListed` 不提供筛选入口，仅用于计算统计概览中的"上市任比例"指标

## 三、页面布局

### 3.1 应用视图

本应用共包含 **2 个视图**：

1. **主搜索页**（默认视图）：左侧筛选面板 + 右侧统计/表格，所有核心功能在此页完成
2. **无权限提示页**：未授权用户访问时显示，提示"您暂无访问权限，请联系管理员"

> 飞书妙搭应用依托飞书账号体系，**无需单独登录页面**，权限控制通过飞书应用后台配置实现。

### 3.2 主搜索页整体布局

```
┌─────────────────────────────────────────────────────────┐
│  顶部标题栏：应用名称 | 数据来源标识 | 当前登录用户        │
├──────────┬──────────────────────────────────────────────┤
│          │  已保存的搜索条件（横向 chip 列表）              │
│  左侧    ├──────────────────────────────────────────────┤
│  筛选    │  统计概览（命中人数 / 涉及公司数 / 上市任比例）    │
│  面板    ├──────────────────────────────────────────────┤
│  (固定   │  搜索结果表格（分页，含排序）                    │
│  268px)  │                                              │
│          │  分页控件                                     │
└──────────┴──────────────────────────────────────────────┘
```

### 3.3 左侧筛选面板

筛选面板固定在左侧，高度占满屏幕，内容可上下滚动，底部操作按钮固定不随滚动消失。

筛选项从上到下排列如下：

| 筛选项 | 组件类型 | 说明 |
|--------|----------|------|
| 公司名称 | 文本输入框 | 模糊匹配公司全称，输入即触发过滤 |
| 高管姓名 | 文本输入框 | 模糊匹配高管姓名，输入即触发过滤 |
| 职位 | 文本输入框 | 模糊匹配职位名称（如输入"CFO"可匹配"首席财务官CFO"） |
| 上市板块 | 多选复选框 | 选项：主板 / 科创板 / 创业板 / 北交所 / 港股 |
| 一级行业 | 多选下拉（含搜索框） | 申万一级行业，共 31 个选项 |
| 二级行业 | 多选下拉（含搜索框） | 申万二级行业；**未选一级行业时显示全部二级行业**；选了一级行业后仅显示该行业下的子行业 |
| 公司市值 | 两个数字输入框（最小值 / 最大值） | 单位：亿（A股按人民币，港股按港元，见下方说明）；留空表示不限 |
| 注册省份 | 多选下拉（含搜索框） | 全国省份/直辖市 |
| 上市年份 | 两个数字输入框（起始年 / 截止年） | 如：1990 ~ 2010；留空表示不限 |
| 候选人出生年份 | 两个数字输入框（起始年 / 截止年） | 如：1960 ~ 1980；留空表示不限 |

**市值筛选框下方需显示说明文字**（灰色小字）：
> "A股单位：亿人民币；港股单位：亿港元，分别计算，不换算汇率。港股约55%记录无市值数据，设置市值范围后这些记录将不显示。"

筛选面板底部固定两个按钮：
- **重置**：清空所有筛选条件，恢复显示全部数据
- **保存条件**：弹出输入框，让用户为当前筛选条件命名，保存后显示在顶部 chip 区域

### 3.4 顶部已保存条件栏

- 横向展示所有已保存的搜索条件，每个条件以蓝色 chip 形式展示
- 每个 chip 显示条件名称，**点击**可一键加载该组筛选条件（覆盖当前所有筛选项）
- 每个 chip 右侧有删除按钮（×），点击删除该条件
- 无保存条件时，显示提示文字"暂无已保存的搜索条件"
- **存储方式**：保存在浏览器 localStorage 中，绑定当前飞书用户 ID，不同用户数据隔离，同一用户同一浏览器下数据持久化

### 3.5 统计概览区

三个指标卡横向排列，右侧为操作按钮区：

| 指标 | 说明 |
|------|------|
| 筛选人数 | 当前结果集中的高管记录总数 |
| 涉及公司 | 当前结果集中涉及的独立公司数（按 `code` 字段去重） |
| 上市任比例 | `isCurrentlyListed = true` 的记录占当前结果集的百分比 |

统计概览区右侧放置**导出 Excel** 按钮。

### 3.6 结果表格

**表头列（固定顺序）：**

| 列名 | 宽度 | 说明 |
|------|------|------|
| 行业 | 100px | 申万一级行业；首列固定，横向滚动时不消失 |
| 代码 | 90px | 股票代码，A 股如 000001，港股如 0700.HK |
| 公司 | 160px | 公司全称 |
| 板块 | 70px | 彩色标签（颜色规范见第七章） |
| 上市 | 60px | 上市年份，支持升/降序排序 |
| 市值 | 90px | 显示格式：¥2,078亿（A股）或 HK$3,450亿（港股）；无数据显示"—"；支持升/降序排序 |
| 姓名 | 70px | 高管姓名 |
| 职位 | 120px | 职位名称 |
| 学历 | 80px | 最高学历 |
| 出生年份 | 80px | 如：1968；无数据显示"—" |
| 任职起始 | 80px | 如：2019-03；无数据显示"—" |

**表格功能：**
- 默认按数据原始顺序排列（不排序）
- 支持点击"市值"列表头和"上市"列表头切换升/降序
- 分页展示，**默认每页 25 条**，支持切换（25 / 50 / 100 条/页）
- 分页控件显示：当前页 / 总页数、上一页/下一页按钮、总条数

## 四、筛选逻辑

### 4.1 基本规则

- 所有筛选条件之间为 **AND 关系**（同时满足全部条件）
- 同一个多选筛选项内部为 **OR 关系**（满足其中任一选项即可）
- 文本输入框为**模糊匹配**，不区分大小写，包含关键词即命中
- 数值范围为**闭区间**（含两端边界值）

### 4.2 二级行业联动规则

- 未选任何一级行业时：二级行业下拉显示**全部**二级行业选项
- 选了一级行业后：二级行业下拉**仅显示**该一级行业下的子行业
- 若已选中某些二级行业，用户又修改了一级行业选择：**不在新一级行业范围内**的已选二级行业自动取消选中

### 4.3 市值筛选规则

- A 股市值单位为亿人民币（CNY），港股市值单位为亿港元（HKD）
- 筛选时 A 股和港股**分别用本币数值比较**，不做汇率换算
- 港股约 55% 的记录市值字段为空值：
  - 用户**未设置**市值范围 → 空值记录**正常显示**
  - 用户**设置了**市值最小值或最大值（任一非空）→ 空值记录**不显示**

### 4.4 搜索性能要求

- 任何筛选条件变更后，结果应在 **1 秒内**刷新
- 实现方式：初始化时将全量数据（约 3.4 万条）一次性加载到前端内存，所有筛选逻辑在前端 JavaScript 中本地执行，不每次请求后端
- 数据加载完成前，表格区域显示 loading 状态，筛选面板不可操作

## 五、导出功能

### 5.1 Excel 导出

- **触发**：点击统计概览区右侧的"导出 Excel"按钮
- **导出范围**：当前筛选条件下的**全部记录**（不仅限于当前页）
- **文件名**：`高管信息导出_YYYYMMDD.xlsx`（日期为导出当天）
- **导出列**（与表格展示列对应，共 12 列）：

| Excel 列名 | 对应字段 | 格式说明 |
|-----------|---------|---------|
| 行业 | industry | 文本 |
| 股票代码 | code | 文本 |
| 公司名称 | company | 文本 |
| 上市板块 | exchange | 文本 |
| 上市年份 | listingYear | 数字 |
| 市值（亿） | marketCap | 数字，保留 1 位小数；无数据留空 |
| 市值币种 | marketCapCurrency | 文本，CNY 或 HKD（**导出时单独一列**，表格展示时合并在市值列内） |
| 姓名 | name | 文本 |
| 职位 | position | 文本 |
| 最高学历 | education | 文本 |
| 出生年份 | birthday | 文本 |
| 任职起始 | tenure | 文本 |

- 若当前筛选结果超过 **5,000 条**，导出前弹出提示："当前结果共 X 条，数据量较大，导出可能需要几秒，是否继续？"，用户确认后再执行导出

### 5.2 管理员数据更新入口（可选功能）

- 入口位置：顶部标题栏右侧，仅对管理员用户可见（非管理员不显示此入口）
- 点击后弹出数据导入对话框，支持上传 JSON 格式数据文件
- 上传过程中显示进度条，完成后提示"成功导入 X 条记录"

## 六、权限控制

### 6.1 访问权限

- 利用**飞书内置应用权限管理**控制访问，无需额外登录页面
- 在飞书应用后台配置可访问的用户或用户组（如指定部门或具体人员）
- 未授权飞书用户访问时，显示"无权限"提示页：内容为"您暂无访问权限，请联系管理员申请"

### 6.2 数据权限

- 所有有访问权限的用户均可查看**全量数据**，不做行级权限区分
- 导出功能对所有有权用户开放

### 6.3 管理员权限（可选）

- 管理员用户在普通权限基础上额外拥有数据更新（导入）权限
- 通过**飞书用户组**区分普通用户与管理员，妙搭应用读取当前用户所属用户组来判断是否为管理员

## 九、技术说明（给妙搭 AI 的实现建议）

### 9.1 数据存储建议

**方案 A（推荐快速验证）：飞书多维表格**
- 将 34,219 条数据导入飞书多维表格（Bitable）
- 妙搭应用直接绑定多维表格作为数据源
- 优点：零额外配置，权限自动与飞书账号打通
- 注意：多维表格单表上限约 5 万行，当前数据量满足要求

**方案 B（推荐长期扩展）：外部数据库 + API**
- 数据存储在 PostgreSQL 等外部数据库
- 妙搭通过 HTTP 连接器调用后端 REST API 获取数据
- 优点：支持更大数据量、更复杂查询、多数据源整合（未来接入 Capital IQ 等）

### 9.2 前端性能建议

- 建议一次性将全量数据加载到前端内存，所有筛选逻辑在前端 JavaScript 中本地执行
- 表格使用虚拟滚动（Virtual Scroll）渲染，避免 3.4 万条 DOM 节点导致页面卡顿
- 初始化时显示加载动画，加载完成再渲染筛选界面

### 9.3 Excel 导出实现建议

- 使用 `SheetJS`（xlsx.js）开源库在前端生成 `.xlsx` 文件并触发下载
- 或调用飞书多维表格的导出 API（如满足字段格式要求）

### 9.4 数据字段映射（JSON → 多维表格字段名）
